CiG: Application Programming Interface

Introduction

#{msgs.title} application evaluates the performance of algorithms to detect cycles in graphs. Results of #{msgs.title} can be accessed programmatically through a RESTful Application Programming Interface (API), The #{msgs.title} API, which is described in this document.

Version

The #{msgs.title} API follows Semantic Versioning for its version numbers. The current API version number is #{msgs.apiversion}.

Base URI

The base URI for Resource Identifiers, on this server, is #{restRoot}

Caching

Currently the #{msgs.title} API does not support caching. However, future versions of the API will support caching mechanisms such as Conditional GETs and Expiration Times.

Authentication and Authorization

Currently the API does not implement authentication or authorization. Note: Authentication mechanisms will be changed in future. So please be prepared to update you application.

Encryption

Currently the API does not enforce encryption. However, once authentication is in place, #{msgs.title} API will force the use of HTTPS protocol; if HTTP is used, it will be redirected over to HTTPS.

CORS

For security reasons, #{msgs.title} API does not support Cross-origin Resource Sharing (CORS).

Error Codes and Responses

The #{msgs.title} API attempts to return appropriate HTTP Status Codes for each request.
