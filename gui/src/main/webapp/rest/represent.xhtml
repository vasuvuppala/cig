Resource Representations

A resource can be manifested or encoded in various formats: binary, XML, JSON, Atom etc. The #{msgs.title} API can represent resources in different formats. Currently it supports JSON and XML representations. The desired representations must be specified through HTTP ACCEPT header:
  • Accept: application/json
  • Accept: application/xml

Example representations for some of the resources are given below.

Note that the resource attribute names match the XML tags and JSON names. However, there can be extraneous attributes in the API's response, and they must be ignored. Only the attributes listed in the Resources Section above form the API.
